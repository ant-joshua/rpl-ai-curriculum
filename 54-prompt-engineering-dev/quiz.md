# Quiz: Prompt Engineering untuk Developer

<div class="quiz">

**1. Apa saja komponen utama dalam anatomi system prompt?**

- [ ] Hanya ROLE dan OUTPUT_FORMAT
- [x] ROLE, CONTEXT, CONSTRAINTS, OUTPUT_FORMAT, RULES, dan SAFETY
- [ ] Hanya CONTEXT dan RULES
- [ ] Persona, API key, dan model name

**2. Teknik prompting apa yang paling efektif untuk memberikan contoh konkret ke LLM?**

- [ ] Zero-shot prompting
- [x] Few-shot prompting (memberikan beberapa contoh input-output)
- [ ] Prompt chaining
- [ ] Keyword stuffing

**3. Apa tujuan chain-of-thought (CoT) prompting?**

- [ ] Membuat output lebih pendek
- [ ] Menghilangkan bias model
- [x] Membuat model berpikir step-by-step sebelum memberikan jawaban untuk task kompleks
- [ ] Mengganti few-shot prompting

**4. Manakah yang termasuk structured output dalam prompting?**

- [ ] Prompt yang ditulis dengan huruf kapital semua
- [x] Meminta output dalam format JSON, XML, atau tabel dengan schema yang jelas
- [ ] Output berupa paragraf panjang
- [ ] Prompt yang tidak memiliki format

**5. Manakah pola prompt (prompt pattern) yang tepat untuk code review task?**

- [ ] Persona pattern tanpa kriteria spesifik
- [x] Code Review Pattern dengan severity levels (CRITICAL, HIGH, MEDIUM, LOW) dan area spesifik
- [ ] Summarization pattern
- [ ] Translation pattern

**6. Dalam production prompting, mengapa versioning prompt penting?**

- [ ] Agar prompt terlihat lebih profesional
- [x] Untuk melacak perubahan, melakukan A/B testing, dan rollback jika performa turun
- [ ] Agar model tidak bisa membaca prompt lama
- [ ] Untuk mempercepat eksekusi prompt
