```markdown
<!-- - Meningkatkan performa API response time 40% dengan implementasi Redis caching -->

### Pendidikan
<!-- === LENGKAPI === -->
```


1. Buat CV ATS-friendly berdasarkan template
2. Analisis keywords dari 3 job description
3. Tulis pengalaman dengan format STAR + impact metrics

**Hint:** ATS-friendly: single column, standard font (Arial/Calibri 10-12pt), section headers jelas, simpan sebagai .docx (bukan PDF). Hindari: tabel, kolom, gambar, icon, header/footer dengan text. Format STAR: Situation → Task → Action → Result + angka. Keywords: baca job description dan cocokkan skill kamu. Tools: Jobscan.co, resume.worded.com.

**Pertanyaan:** Siapkan jawaban behavioral interview dengan metode STAR:

```markdown
# === LENGKAPI: STAR answers ===

## Pertanyaan 1: "Ceritakan tentang project tersulit yang pernah kamu kerjakan"
### Situation (Situasi)
<!-- === LENGKAPI: Deskripsikan konteks project === -->
### Task (Tugas)
<!-- === LENGKAPI: Apa tanggung jawab kamu specifically === -->
### Action (Tindakan)
<!-- === LENGKAPI: Langkah konkret yang kamu ambil === -->
### Result (Hasil)
<!-- === LENGKAPI: Outcome dengan angka/fakta === -->

## Pertanyaan 2: "Pernah conflict dengan team member? Gimana cara handle?"
<!-- === LENGKAPI: Tulis STAR answer === -->

## Pertanyaan 3: "Gimana cara kamu belajar teknologi baru?"
<!-- === LENGKAPI: Tulis STAR answer dengan contoh konkret === -->
```

```
# === LENGKAPI: Technical interview prep ===
# 1. Algoritma & Data Structure yang sering ditanyakan:
#    - ?
#    - ?
#    - ?
# 2. System design untuk junior:
#    - ?
#    - ?
# 3. Bahasa pemrograman (JavaScript/TypeScript):
#    - ?
#    - ?
```

1. Siapkan 5 STAR answers untuk pertanyaan behavioral umum
2. Review struktur data dan algoritma dasar
3. Siapkan whiteboard coding untuk problem sederhana

**Hint:** STAR: Situation (latar belakang) → Task (tanggung jawab kamu) → Action (langkah kamu) → Result (hasil dengan angka). Behavioral questions: toughest project, conflict resolution, failure experience, teamwork, learning new tech. Technical prep: FizzBuzz, palindrome, anagram, two sum, reverse linked list. System design: design URL shortener, design chat app. Jujur kalau tidak tahu — tunjukkan cara berpikir.

### 8. Tech Ethics & Professionalisme
**Pertanyaan:** Analisis skenario etika di dunia kerja teknologi:

```markdown
# === LENGKAPI: Ethical scenarios ===

## Skenario 1: Data User
Kamu diminta manager untuk menyimpan data user (termasuk password) di log untuk debugging.
# Pertanyaan: 
# 1. Apa masalah etika dari situasi ini?
# 2. Apa yang akan kamu lakukan?
# 3. Alternatif solusi apa yang bisa diajukan?
# === LENGKAPI: Jawaban ===

## Skenario 2: Deadline vs Quality
Tim dikejar deadline, lead developer suruh cut corners: skip testing, hardcode credentials.
# === LENGKAPI: Analisis dan solusi ===

## Skenario 3: Intellectual Property
Kamu nemu kode proyek lama di laptop pribadi yang bisa dipake untuk freelance client baru.
# === LENGKAPI: Analisis dan solusi ===

## Skenario 4: Overwork Culture
Perusahaan expect kerja lembur tiap hari tanpa bayaran tambahan.
# === LENGKAPI: Analisis dan solusi ===
```

```
# === LENGKAPI: Professional code of conduct ===
# Buat personal code of conduct sebagai developer:
# 1. ?
# 2. ?
# 3. ?
# 4. ?
# 5. ?
```

1. Analisis setiap skenario etika
2. Tentukan tindakan yang tepat untuk tiap skenario
3. Buat personal code of conduct
4. Diskusikan dengan teman atau mentor

**Hint:** Prinsip etika: (1) Jangan cause harm, (2) Hormati privacy user, (3) Jujur tentang kemampuan, (4) Credit orang lain untuk work mereka, (5) Laporkan unethical practice. Data privacy: hash password, jangan log sensitive data. IP: kode perusahaan adalah milik perusahaan. Overwork: komunikasi dengan manager, track work hours. Kalau lingkungan toxic: prepare exit strategy sambil tetap profesional.
