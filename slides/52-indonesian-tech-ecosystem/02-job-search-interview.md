# Sesi 2: Job Search & Interview — Dapatkan Pekerjaan Impian di Teknologi

**Durasi:** 90 menit

---

## Tujuan Sesi

Setelah sesi ini, peserta mampu:
- Menyusun CV ATS-friendly yang lolos screening system
- Mengoptimalkan profil LinkedIn untuk dilirik recruiter
- Menggunakan job portal secara efektif (Glints, Karir.com, LinkedIn, Kalibrr)
- Menghadapi behavioral interview dengan metode STAR
- Mempersiapkan technical test, coding challenge, dan take-home test

---

## 1. CV ATS-Friendly (20 menit)

### Apa itu ATS?

**ATS (Applicant Tracking System)** adalah software yang digunakan HRD untuk menyaring CV secara otomatis. Hampir semua perusahaan teknologi di Indonesia (Gojek, Tokopedia, Traveloka, startup) pakai ATS.

**Statistik:** 75% CV tidak lolos ATS screening.

### Cara Kerja ATS

```
CV Kamu → ATS Parser → Data terstruktur:
- Nama
- Kontak
- Pengalaman
- Pendidikan
- Skills → Match % → Job Description
- Keywords
```

Jika keyword di CV tidak match dengan JD → otomatis rejected.

### Format CV yang ATS-Friendly

| ✅ Lolos ATS | ❌ Gagal ATS |
|-------------|-------------|
| PDF standar (text-based) | PDF hasil scan/foto |
| DOCX (Word) | JPG/PNG |
| Format clean, 1 kolom | 2-3 kolom, tabel kompleks |
| Font standar (Arial, Calibri) | Font unik/dekoratif |
| Bullet points sederhana | Ikon, symbol, chart |
| Header standar | Text box, word art |

### Struktur CV ATS

```
[HEADER]
Nama Lengkap
Email | No Telepon | LinkedIn URL | Portfolio/GitHub
Lokasi: Kota, Indonesia

[RINGKASAN PROFESIONAL — 2-3 kalimat]
[Posisi] dengan [X] tahun pengalaman di [bidang].
Spesialisasi: [skill utama 1], [skill utama 2], [skill utama 3].
Target: [posisi yang dicari].

[PENGALAMAN KERJA]
[Perusahaan] | [Lokasi] | [Bulan Tahun] - [Bulan Tahun]
[Jabatan]
- [Capaian/hasil, gunakan angka]
- [Capaian/hasil, gunakan angka]
- [Capaian/hasil, gunakan angka]

[PENDIDIKAN]
[Universitas] | [Jurusan] | [Tahun Lulus]

[SKILL]
- Technical: [skill 1], [skill 2], [skill 3], [skill 4], [skill 5]
- Soft Skills: [skill 1], [skill 2], [skill 3]

[SERTIFIKASI / BAHASA]
- [Nama sertifikasi] — [Tahun]
- [Nama sertifikasi] — [Tahun]
```

### Keyword Optimization

**Langkah-langkah:**

1. Buka job description yang dituju
2. Identifikasi keyword utama:
   - Technical skills: React, Node.js, TypeScript, PostgreSQL, Docker
   - Tools: Git, Jira, Figma, Postman
   - Soft skills: kolaborasi, problem-solving, komunikasi
3. Masukkan keyword secara natural ke CV
4. Gunakan kata kerja aktif: *Mengembangkan, Merancang, Mengoptimalkan, Memimpin*

**Contoh CV Bullet Point:**

| ❌ Biasa | ✅ Hasil & Angka |
|----------|-----------------|
| Membangun website company profile | Mengembangkan website company profile dengan React.js yang meningkatkan loading speed 40% |
| Bertanggung jawab atas API | Mendesain dan mengimplementasikan 15 REST API endpoints untuk sistem payment |
| Maintenance aplikasi | Mengurangi bug production sebesar 60% dalam 3 bulan melalui code review dan testing |

### File Type

- **PDF**: Format terbaik — mengunci layout, text-based (bukan scan)
- **DOCX**: Alternatif — bisa diedit ATS, tapi layout bisa berubah
- **HINDARI**: JPG, PNG, Canva export (sering gagal di-parsing ATS)

### Tools CV ATS Check

- [Jobscan.co](https://www.jobscan.co) — scan CV vs JD
- [ResumeWorded.com](https://resumeworded.com) — feedback otomatis
- [Skillroads.com](https://skillroads.com) — AI resume builder

## 2. LinkedIn Optimization (20 menit)

LinkedIn adalah platform #1 untuk rekrutmen teknologi di Indonesia. Hampir semua tech recruiter mencari kandidat di LinkedIn.

### Headline Formula

Jangan pakai headline default "*[Posisi] di [Perusahaan]*".

**Formula:**

```
[Role] | [Spesialisasi 1] + [Spesialisasi 2] | [Nilai Jual Unik]
```

**Contoh:**

- *Frontend Developer | React + TypeScript | Membangun Web App Performa Tinggi*
- *Backend Engineer | Node.js + Golang | Microservices & API Design*
- *Data Analyst | SQL + Python + Tableau | Mengubah Data Menjadi Insight Bisnis*
- *UI/UX Designer | Figma + Design System | Menciptakan Pengalaman Digital yang Intuitif*

### About Section

Template:

```
[Pembuka — hook dalam 2 kalimat pertama]

Saya adalah [role] dengan [X] tahun pengalaman di [bidang].
Spesialisasi saya meliputi [skill 1], [skill 2], dan [skill 3].

[Nilai jual — apa yang membuatmu berbeda]
✅ [Pencapaian 1 dengan angka]
✅ [Pencapaian 2 dengan angka]
✅ [Pencapaian 3 dengan angka]

[Tools & teknologi yang digunakan]
Tech stack: [list tools]

[Tertarik bekerja sama / target karir]
Saat ini terbuka untuk [opportunity yang dicari]. 
Yuk connect dan diskusi!
```

### Experience Section

- Tulis seperti CV ATS — bullet points dengan angka
- Sertakan media (screenshot, link, slide) di setiap posisi
- Minta rekomendasi dari atasan/rekan setelah resign

### Skills & Endorsements

- Pilih 10 skill paling relevan
- Urutkan berdasarkan prioritas
- Minta endorsements dari koneksi
- Skill dengan banyak endorsement meningkatkan search ranking

### Recommendations

- Minta minimal 3 rekomendasi dari:
  - Mantan atasan (tentang performa kerja)
  - Rekan tim (tentang kolaborasi)
  - Klien (tentang hasil kerja)

### Aktivitas yang Meningkatkan Visibility

| Aktivitas | Dampak |
|-----------|--------|
| Posting 1-2x/minggu tentang tech | Meningkatkan profil views |
| Comment di post recruiter | Muncul di timeline recruiter |
| Follow company halaman target | Notifikasi saat ada lowongan |
| Join LinkedIn Groups (tech Indonesia) | Networking |
| Share portfolio/project | Bukti kompetensi |

### Open to Work

- Nyalakan `#OpenToWork` setting (bisa private, hanya recruiter lihat)
- Setel posisi, lokasi, dan tipe pekerjaan yang diinginkan
- Isi preferred salary range (realistis)

## 3. Job Portal (10 menit)

### Portal Utama

| Portal | Keunggulan | Kekurangan |
|--------|-----------|------------|
| **LinkedIn** | Networking + apply langsung | Kompetisi tinggi |
| **Glints** | Cocok untuk fresh grad & mid-level | Lowongan senior terbatas |
| **Karir.com** | Banyak startup Indonesia | UI kurang modern |
| **Kalibrr** | Banyak perusahaan besar | Proses panjang |
| **Jobstreet** | Legacy portal, banyak lowongan | Filter kurang baik |
| **Tech in Asia Jobs** | Startup dan tech company | Jumlah lowongan terbatas |

### Tips Apply

1. **Apply early** — dalam 48 jam pertama postingan
2. **Customize CV** — sesuaikan kata kunci per JD
3. **Cover letter singkat** — 3-4 paragraf targeted
4. **Follow-up** — setelah 5-7 hari, kirim pesan ke recruiter di LinkedIn
5. **Track applications** — pakai spreadsheet

### Spreadsheet Tracker

| No | Perusahaan | Posisi | Portal | Tanggal Apply | Status | Follow-up |
|----|-----------|--------|--------|---------------|--------|-----------|
| 1 | Gojek | Backend Eng | LinkedIn | 05/01 | Applied | 12/01 |
| 2 | Tokopedia | FE Dev | Glints | 06/01 | Screening | - |

### Networking untuk Referral

Referral punya chance 10x lebih besar dibanding apply biasa.

**Cara minta referral:**
1. Cari orang di perusahaan target via LinkedIn
2. Kirim pesan sopan:
   > "Halo [Nama], saya [Nama]. Saya tertarik dengan posisi [X] di [Perusahaan]. Melihat background kamu sebagai [posisi] di sana, apakah kamu ada waktu 5 menit untuk diskusi tentang budaya kerja dan tim di [Perusahaan]?"
3. Jika respon positif, tanya tentang proses interview
4. Jika sudah apply, minta di-refer
5. Jangan lupa terima kasih — treat kopi atau makan siang

## 4. Interview (25 menit)

### Behavioral Interview — Metode STAR

**STAR = Situation, Task, Action, Result**

```
S — Situation: Konteks situasi
T — Task: Tugas/tantangan yang harus diselesaikan
A — Action: Tindakan spesifik yang kamu lakukan
R — Result: Hasil yang dicapai (pakai angka)
```

**Contoh STAR — "Ceritakan tentang konflik di tim"**

| Komponen | Jawaban |
|----------|---------|
| **S** | Di project e-commerce sebelumnya, tim frontend dan backend sering berbeda pendapat tentang API contract |
| **T** | Deadline mepet, perlu standarisasi komunikasi API antar tim |
| **A** | Saya inisiatif bikin API documentation di Swagger, jadwal sync daily 15 menit, dan Discord channel khusus API discussion |
| **R** | Conflict turun 80%, delivery on time, dan API contract documentation jadi standar tim |

**Prepare jawaban STAR untuk pertanyaan ini:**

1. Ceritakan tentang diri kamu
2. Kegagalan terbesar dan bagaimana mengatasinya
3. Konflik dengan rekan tim
4. Project paling challenging
5. Waktu kamu harus belajar teknologi baru cepat
6. Pencapaian yang paling dibanggakan
7. Kenapa kamu mau bekerja di sini?

### Technical Test — Coding Challenge

**Format umum:**

| Jenis | Durasi | Platform |
|-------|--------|----------|
| Online Assessment (OA) | 60-120 menit | HackerRank, Codility, Alva |
| Live Coding | 45-60 menit | CoderPad, Google Docs |
| Pair Programming | 60 menit | Zoom + IDE sharing |

**Tips:**

1. **Baca soal 2x** — pahami sebelum coding
2. **Tanya clarifying questions** — jangan asumsi
3. **Bahas approach dulu** — verbalize thinking process
4. **Write clean code** — bukan cuma solve
5. **Test dengan contoh** — tunjukkan debug mindset
6. **Optimize setelah working** — jangan premature optimization
7. **Time management** — jika stuck >15 menit, skip

**Topik yang sering muncul:**

| Level | Topik |
|-------|-------|
| Junior | Array, string, looping, sorting, hashmap |
| Mid | Two pointers, sliding window, recursion, tree traversal, BFS/DFS |
| Senior | Dynamic programming, graph, system design, concurrency |

### System Design Interview

Untuk posisi mid-senior dan lead.

**Framework menjawab:**

```
1. Clarify requirements (functional + non-functional)
2. Estimate scale (users, data volume, QPS)
3. High-level design (components diagram)
4. Database schema (SQL vs NoSQL)
5. API design (endpoints)
6. Deep dive (bagian paling menarik/challenging)
7. Trade-offs (pros/cons dari pilihan arsitektur)
```

**Topik yang sering:**

- Design URL shortener (TinyURL)
- Design chat system (WhatsApp)
- Design e-commerce platform
- Design social media feed
- Design payment system
- Design ride-sharing (Gojek/Grab)

### Persiapan Sebelum Interview

```
✅ Riset perusahaan — produk, misi, tech stack, recent news
✅ Pahami job description — skill apa yang mereka cari
✅ Siapkan 3 pertanyaan cerdas untuk pewawancara
✅ Cek koneksi internet, kamera, microphone
✅ Siapkan environment coding (IDE ready, screen sharing ready)
✅ Tidur cukup malam sebelumnya
✅ Datang 5-10 menit lebih awal
```

## 5. Take-Home Test (10 menit)

### Karakteristik

- Diberikan setelah screening CV
- Waktu: 2-7 hari
- Deliverable: repo GitHub + README + demo (jika ada)
- Tujuan: lihat cara kerja, code quality, problem-solving

### Strategi

| Fase | Aktivitas |
|------|-----------|
| **Day 1** | Baca soal, clarify, research, rencanakan arsitektur |
| **Day 2-3** | Coding fitur inti, setup testing |
| **Day 4** | Polish, handle edge cases, refactor |
| **Day 5** | Dokumentasi, README, deploy demo |
| **Final** | Review & submit sebelum deadline |

### Yang Dinilai

| Aspek | Bobot |
|-------|-------|
| Functionality — apakah app berjalan | 30% |
| Code quality — clean code, naming, structure | 25% |
| Testing — unit test, integration test | 20% |
| Documentation — README, comments, commit messages | 15% |
| Extras — deployment, CI/CD, bonus features | 10% |

### Checklist Sebelum Submit

```
- [ ] App berjalan tanpa error
- [ ] README: deskripsi, cara run, tech stack, asumsi
- [ ] Unit test minimal untuk logic utama
- [ ] Error handling untuk edge cases
- [ ] Tidak ada code yang di-comment out
- [ ] Tidak ada console.log atau debug code
- [ ] Commit messages rapi (conventional commits)
- [ ] Tidak ada credentials atau .env di repo
- [ ] Jika ada demo: link working di README
```

## 6. Latihan Praktik (5 menit)

### Latihan 1: Bikin CV ATS-Friendly
1. Ambil CV yang sudah ada (atau buat dari template)
2. Struktur ulang sesuai format ATS-friendly
3. Pilih 1 job description dari Glints/Kalibrr/LinkedIn
4. Optimalkan keyword di CV berdasarkan JD tersebut
5. Export sebagai PDF
6. Scan dengan Jobscan.co atau ResumeWorded.com
7. Iterasi hingga match score > 70%

### Latihan 2: Optimasi LinkedIn
1. Update headline menggunakan formula
2. Tulis ulang About section
3. Update experience dengan bullet points berisi angka
4. Minta 1 endorsement dari teman
5. Atur #OpenToWork setting
6. Cari 3 perusahaan target dan follow halaman mereka

### Latihan 3: Simulasi Interview
- Siapkan 3 jawaban STAR untuk pertanyaan behavioral umum
- Selesaikan 1 soal coding easy-medium di HackerRank
- Siapkan 1 pertanyaan untuk pewawancara (untuk setiap perusahaan target)
