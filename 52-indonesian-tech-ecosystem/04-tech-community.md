### Cara Bergabung

1. **Cari link invite** — cek GitHub community pages, Twitter, LinkedIn
2. **Perkenalkan diri** — jangan silent reader terus
3. **Ikuti aturan** — baca pinned messages, rules
4. **Aktif bertanya & menjawab** — berkontribusi sejak hari pertama

### Etika di Komunitas

| ✅ Lakukan | ❌ Jangan |
|-----------|----------|
| Perkenalkan diri dengan sopan | Spam promosi/link afiliasi |
| Baca FAQ sebelum bertanya | Tanya yang sudah dijawab di pinned |
| Gunakan bahasa yang sopan | Toxic, nyinyir, flame war |
| Bantu jawab pertanyaan orang lain | Hanya numpang promo/lowongan |
| Follow up jika ada solusi | Ghosting setelah dibantu |

### Case Study: Dari Silent Reader ke Kontributor Aktif

> **Rina** — frontend developer junior, bergabung dengan ReactJS Indonesia. Awalnya hanya baca-baca. Suatu hari jawab pertanyaan tentang useState. Semakin aktif, dikenal sebagai "si rajin bantu." Dapat tawaran freelance dari anggota grup. Setahun kemudian diminta jadi speaker di meetup. Sekarang jadi salah satu moderator grup.

## 2. Meetup & Workshop (15 menit)

### Jenis Event

| Jenis | Durasi | Format | Cocok Untuk |
|-------|--------|--------|-------------|
| **Tech Talk** | 1-2 jam | Presentasi + Q&A | Belajar teknologi baru |
| **Workshop** | 3-8 jam | Hands-on coding | Praktek langsung |
| **Study Jam** | 4-16 minggu | Sesi rutin mingguan | Persiapan sertifikasi |
| **Conference** | 1-3 hari | Multi-track, keynote | Networking, high-level insight |
| **Lightning Talk** | 5-15 menit | Presentasi singkat | Latihan public speaking |

### Platform Cari Event

| Platform | Keunggulan |
|----------|-----------|
| **Eventbrite** | Banyak event tech, RSVP mudah |
| **Meetup.com** | Komunitas internasional & lokal |
| **LinkedIn Events** | Event dari perusahaan & komunitas |
| **Instagram** | Follow komunitas untuk update event |
| **Telegram Groups** | Announcement event dari komunitas |

### Cara Mendapatkan Manfaat Maksimal dari Meetup

**Sebelum event:**
```
✅ Daftar (RSVP) — jangan no-show
✅ Baca topik dan speaker profile
✅ Siapkan 1-2 pertanyaan
✅ Update LinkedIn status: "Going to [event]"
✅ Bawa kartu nama (digital atau fisik)
```

**Saat event:**
```
✅ Datang tepat waktu
✅ Kenalan 3-5 orang baru:
   - "Halo, saya [nama], [role] di [perusahaan/aktivitas]"
   - "Baru pertama kali ke meetup ini?"
   - "Tertarik dengan topik apa?"
✅ Ikut Q&A — ajukan pertanyaan berarti
✅ Catat poin penting
✅ Follow social media speaker & peserta baru
```

**Setelah event:**
```
✅ Kirim LinkedIn request ke orang baru:
   > "Senang kenalan di [event] tadi. Semoga bisa diskusi lagi!"
✅ Kirim feedback ke penyelenggara
✅ Posting refleksi di LinkedIn:
   > "Baru selesai dari [event]. Insight favorit saya: [key takeaway]"
✅ Apply ilmu yang didapat
```

### Jadi Speaker

**Kenapa harus jadi speaker?**
- Personal branding
- Networking dengan expert
- Dianggap authoritatif di bidang
- Membuka opportunity (kerja, freelance, consulting)

**Cara mulai:**
1. Mulai dari lightning talk (5-10 menit)
2. Topik: project yang pernah dikerjakan, kesalahan yang pernah dibuat, tips & trik
3. Daftar sebagai speaker di komunitas
4. Kirim proposal ke conference (CFP — Call for Proposals)

## 3. Hackathon (15 menit)

### Apa Itu Hackathon?

**Hackathon** = Hack + Marathon — kompetisi coding 24-72 jam untuk membuat produk (biasanya prototype) berdasarkan tema tertentu.

| Aspek | Deskripsi |
|-------|-----------|
| **Durasi** | 24 jam (weekend), 48 jam, atau 72 jam |
| **Tim** | 2-5 orang (biasanya) |
| **Output** | Working prototype + presentasi (pitch) |
| **Hadiah** | Uang tunai, device, mentorship, investor access |
| **Tema** | Fintech, Healthtech, Edtech, Smart City, ESG |

### Hackathon Populer di Indonesia

| Nama | Penyelenggara | Hadiah |
|------|--------------|--------|
| **Indigo Hackathon** | Telkom | Rp 50-100 juta |
| **Hackathon.id** | Komunitas | Device + sertifikat |
| **BNI Hackathon** | BNI | Rp 75 juta |
| **Shopee Code League** | Shopee | Device + interview |
| **Google Solution Challenge** | Google Developer | $3,000 + mentorship |
| **TreeHacks** (Stanford) | Stanford University | $10,000+ |

### Strategi Hackathon

#### Fase 1: Ideasi (3-4 jam pertama)
```
✅ Pahami tema & kriteria penilaian
✅ Brainstorm — 10 ide, pilih 1 terbaik
✅ Validasi ide:
   - Apakah solve real problem?
   - Feasible dalam 24 jam?
   - Unik / beda dari peserta lain?
✅ Tentukan tech stack — pakai yang paling dikuasai
```

#### Fase 2: Eksekusi (16-20 jam)
```
✅ Jangan over-engineering — MVP dulu
✅ Bagi tugas tim berdasarkan kekuatan
✅ Setup: repo, CI/CD, deploy early
✅ Fitur inti selesai dalam 12 jam pertama
✅ Fitur bonus setelah inti stabil
✅ Istirahat bergiliran — jangan burnout
```

#### Fase 3: Finalisasi (4-6 jam terakhir)
```
✅ Bug fixing — critical bugs only
✅ Polish UI — first impression penting
✅ Siapkan demo — real working app
✅ Siapkan slide pitching (3-5 slide)
✅ Latihan pitching — 3 menit, jelas, impactful
✅ Sleep jangan sampai skipped
```

### Pitching

**Struktur Pitch 3 Menit:**

| Waktu | Konten |
|-------|--------|
| **0:00-0:30** | Problem — apa masalah yang dipecahkan? |
| **0:30-1:00** | Solution — produk kita sebagai jawaban |
| **1:00-1:30** | Demo — tunjukkan produk LIVE (bukan slide) |
| **1:30-2:00** | Tech stack & arsitektur singkat |
| **2:00-2:30** | Business value / impact |
| **2:30-3:00** | Next steps & closing |

**Tips pitching:**
- Demo > slide — juri mau lihat produk jalan
- Cerita (storytelling) — bikin juri peduli
- Jangan jelasin coding detail — fokus ke user impact
- Tim kompak — semua paham produk
- Latihan minimal 3x sebelum actual pitch

### Yang Dinilai Juri

| Kriteria | Bobot |
|----------|-------|
| **Problem relevance** — apakah solve real problem? | 25% |
| **Completeness** — seberapa working prototype-nya? | 25% |
| **Technical difficulty** — seberapa complex tech-nya? | 20% |
| **Innovation** — seberapa unik solusinya? | 15% |
| **Presentation** — seberapa meyakinkan pitch-nya? | 15% |

## 4. Open Source (20 menit)

### Kenapa Open Source?

- **Portfolio proof** — kode kamu dilihat ribuan orang
- **Belajar dari best practice** — baca kode senior engineer
- **Networking** — dikenal oleh maintainer project besar
- **Karir** — banyak perusahaan tech lihat kontribusi GitHub
- **Skill** — code review, testing, documentation, communication

### Memulai Open Source

#### 1. Persiapan Awal
```
✅ Buat GitHub profile yang rapi:
   - Foto profile, bio, link portfolio
   - Pin 3-6 repo terbaik
   - README profil (bisa pakai template)
✅ Pahami Git dasar:
   - Fork, clone, branch, commit, push
   - Pull request, rebase, resolve conflict
```

#### 2. Cari Project yang Tepat

| Platform | Tips Mencari |
|----------|-------------|
| **GitHub Explore** | Cari berdasarkan bahasa, topic label |
| **Good First Issues** | Filter: `good first issue`, `help wanted`, `beginner friendly` |
| **First Timers Only** | List issue khusus pemula |
| **Up For Grabs** | Project yang welcome kontributor baru |
| **CodeTriage** | Subscribe issue dari project favorit |

#### 3. Kontribusi Pertama

**Step-by-step:**

1. Pilih project — mulai dari yang kamu pakai sehari-hari (React, Laravel, Python library)
2. Baca CONTRIBUTING.md — aturan main kontribusi
3. Cari issue `good first issue` — jangan ambil yang kompleks
4. Comment: *"I'd like to work on this"* — biar tidak clash dengan contributor lain
5. Fork repo, clone, buat branch
6. Kerjakan issue
7. Commit dengan format Convention: `feat: add login validation`, `fix: resolve memory leak`, `docs: update readme`
8. Push, buat Pull Request
9. Bersabar — maintainer butuh waktu untuk review

#### 4. Jenis Kontribusi

**Tidak harus coding!**

| Jenis | Contoh | Skill yang Diperlukan |
|-------|--------|----------------------|
| **Dokumentasi** | README, fixing typo, tutorial | Bahasa Inggris yang baik |
| **Bug fix** | Fix bug kecil, edge cases | Programming |
| **Fitur baru** | Tambah fitur sesuai roadmap | Programming level tinggi |
| **Code review** | Review PR orang lain | Pemahaman kode yang baik |
| **Testing** | Tambah unit test, integration test | Testing framework |
| **Translation** | Terjemahkan dokumentasi | Bilingual |
| **Issue triage** | Reproduce bug, kategorisasi | Debug skill |
| **Community** | Bantu di forum, jawab pertanyaan | Komunikasi |

### PR Pertama Checklist

```
[ ] Baca CONTRIBUTING.md
[ ] Cek issue: belum ada yang assign
[ ] Fork dari repo utama (bukan fork punya orang lain)
[ ] Branch naming: `fix/issue-42` atau `feat/add-dark-mode`
[ ] Commit message: format conventional commits
[ ] Testing: pastikan tidak broken
[ ] PR description: jelaskan apa yang diubah dan kenapa
[ ] Screenshot/GIF: untuk UI changes
[ ] Referensi: `Closes #42` — auto-close issue
[ ] Sabar: maintainer mungkin sibuk
```

## 5. Tech Blog (15 menit)

### Kenapa Nulis Tech Blog?

| Manfaat | Dampak |
|---------|--------|
| **Memperdalam pemahaman** | Menulis = belajar 2x lebih efektif |
| **Personal branding** | Dikenal sebagai expert di topik tertentu |
| **Portfolio** | Bisa ditunjukkan ke recruiter |
| **Passive networking** | Orang lain reach out karena artikel |
| **Koneksi baru** | Comment, feedback, diskusi |
| **Peluang karir** | Ditawari freelance, speaker, bahkan kerja |

### Platform

| Platform | Keunggulan | Kekurangan | SEO |
|----------|-----------|------------|-----|
| **dev.to** | Komunitas developer global, free | Bahasa Inggris | Bagus |
| **Medium** | Mudah dipakai, built-in audience | Limited customization | Bagus |
| **Personal blog** | Milik sendiri, brand sendiri | Setup repot | Tergantung domain |
| **Hashnode** | Dev-focused, free + custom domain | Komunitas lebih kecil | Cukup |
| **LinkedIn Articles** | Eksposur ke koneksi langsung | Format terbatas | Sedang |
| **Notion / GitHub Pages** | Gratis, sederhana | Kurang discovery | Rendah |

|----------|-------------|

### Tips Menulis

```
✅ Pilih niche — jangan nulis semua topik
✅ Judul yang clickable — tapi jangan clickbait
✅ Sertakan kode yang bisa di-copy
✅ Screenshot, GIF, diagram — visual > teks
✅ Cerita — bikin personal, relatable
✅ Konsisten — 1 artikel per 1-2 minggu
✅ Promosi — share ke LinkedIn, Twitter, komunitas
✅ Engage — balas comment, diskusi dengan pembaca
✅ Update artikel lama dengan info baru
✅ Jangan plagiat — tulis dengan kata-kata sendiri
```

## 6. Latihan Praktik (5 menit)

### Latihan 1: Cari & Gabung Komunitas

1. Cari 3 komunitas teknologi Indonesia yang relevan dengan minat kamu
2. Gabung ke Telegram / Discord group mereka
3. Perkenalkan diri dalam 1-2 kalimat
4. Cari 1 pertanyaan yang bisa kamu jawab — bantu orang lain
5. Catat di jurnal: komunitas apa, apa yang dipelajari, siapa yang dikenal

### Latihan 2: Rencana Partisipasi 6 Bulan

Buat rencana partisipasi komunitas untuk 6 bulan ke depan:

```
--- RENCANA PARTISIPASI KOMUNITAS ---

Nama: [Nama]
Bulan mulai: [Bulan]

KOMUNITAS YANG DIIKUTI:
1. [Nama komunitas] — untuk [alasan]
2. [Nama komunitas] — untuk [alasan]
3. [Nama komunitas] — untuk [alasan]

TARGET PARTISIPASI:
✅ Hadiri minimal [X] meetup
✅ Bantu jawab [X] pertanyaan di grup per bulan
✅ Buat [X] kontribusi open source
✅ Tulis [X] artikel tech blog
✅ Ikut [X] hackathon
✅ Jadi speaker di [X] lightning talk

MILESTONE PER BULAN:
Bulan 1: [Gabung grup, observasi, jawab 1 pertanyaan/minggu]
Bulan 2: [Hadiri 1 meetup, kenalan 3 orang]
Bulan 3: [Buat PR pertama ke open source]
Bulan 4: [Tulis artikel pertama]
Bulan 5: [Ikut hackathon / jadi volunteer event]
Bulan 6: [Evaluasi, target tahun depan]
```

### Latihan 3: Open Source First Contribution

1. Cari 1 issue `good first issue` di GitHub
2. Fork project tersebut
3. Buat branch, kerjakan issue
4. Kirim Pull Request
5. Dokumentasikan prosesnya di jurnal

### Latihan 4: Draft Tech Blog Artikel

1. Pilih topik yang kamu kuasai
2. Ikuti template artikel
3. Tulis minimal 500 kata
4. Sertakan minimal 1 code block dan 1 screenshot
5. Simpan draft, review setelah 1 hari, baru publish
