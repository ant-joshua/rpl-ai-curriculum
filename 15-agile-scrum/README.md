# 15. Agile & Scrum untuk Proyek Tim

> **Level:** 🌱 Beginner  
> **Jam:** 6 (3 sesi × 2 JP)  
> **Prasyarat:** Git dasar (commit, push, pull), kerja kelompok  
> **Output:** Sprint plan + GitHub Project untuk proyek final  

## Tujuan Pembelajaran

Setelah modul ini, kamu bisa:
- Bedain Agile vs Waterfall dan kapan pakai masing-masing
- Jelasin 3 peran Scrum (PO, SM, Dev) dan tugasnya
- Jalani 4 upacara Scrum (planning, standup, review, retro)
- Tulis User Story pakai format INVEST
- Estimasi task pakai story point / planning poker
- Setup GitHub Projects buat sprint board
- Baca & interpretasi burndown chart
- Buat Sprint Plan untuk proyek final kelas

## AI Prompt Exercises

Sepanjang modul, latihan pake AI:

- "Buatkan 5 user story untuk aplikasi [nama aplikasi] pakai format 'Sebagai... saya ingin... agar...'"
- "Kasih saya contoh sprint goal yang bagus untuk fitur login, register, dan profile"
- "Evaluasi user story ini: [tempel story]. Apakah sudah INVEST? Kasih saran perbaikan."
- "Simulasikan daily standup: saya jadi Budi, kamu jadi SM. Tanya saya 3 pertanyaan standup."
- "Buatkan template retrospective pake format Start/Stop/Continue untuk tim 4 orang"
- "Bantu saya breakdown user story ini jadi task kecil: [tempel story]"
- "Estimasi dengan planning poker: ceritain cara mainnya terus kasih contoh skenario"
