# Quiz: Authentication & Identity

<div class="quiz">

**1. Alur OAuth 2.0 Authorization Code Grant melibatkan:**

- [ ] Hanya client ID dan client secret
- [x] Authorization code yang ditukar dengan access token, melibatkan frontend dan backend
- [ ] Username dan password dikirim langsung ke third-party
- [ ] Token di-generate tanpa interaksi user

**2. JWT (JSON Web Token) terdiri dari tiga bagian:**

- [x] Header, Payload, Signature
- [ ] Header, Body, Footer
- [ ] Token, Secret, Hash
- [ ] Public Key, Private Key, Certificate

**3. OpenID Connect (OIDC) adalah lapisan di atas OAuth 2.0 yang:**

- [ ] Menambahkan fitur refresh token
- [x] Menyediakan identity layer — verifikasi identitas user via ID Token
- [ ] Mengganti access token dengan session cookie
- [ ] Menghilangkan kebutuhan akan client secret

**4. TOTP (Time-based One-Time Password) biasanya digunakan untuk:**

- [ ] Login tanpa password
- [x] Two-factor authentication — kode 6 digit yang berubah setiap 30 detik
- [ ] Reset password
- [ ] Enkripsi data di database

**5. Refresh token digunakan ketika:**

- [x] Access token expired, untuk mendapatkan access token baru tanpa login ulang
- [ ] User pertama kali login
- [ ] Password user berubah
- [ ] Aplikasi di-restart

**6. SSO (Single Sign-On) memungkinkan:**

- [ ] Satu password untuk semua akun berbeda
- [x] Login satu kali ke identity provider untuk mengakses multiple aplikasi
- [ ] Login tanpa password sama sekali
- [ ] Satu akun bisa dipakai banyak orang bersamaan
