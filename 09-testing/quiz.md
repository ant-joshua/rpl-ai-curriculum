# Quiz: Testing & CI/CD

> Jawab pertanyaan di bawah dengan klik pilihan yang benar.

<div class="quiz">

**1. Unit test menguji...**

- [ ] Seluruh aplikasi dari UI sampai database
- [x] Satu unit fungsi/komponen secara terisolasi
- [ ] Hanya tampilan UI
- [ ] Performa server

**2. Library testing populer untuk JavaScript/TypeScript?**

- [x] Jest / Vitest
- [ ] Django Test
- [ ] PHPUnit
- [ ] RSpec

**3. Mock dalam testing digunakan untuk...**

- [ ] Menulis dokumentasi
- [x] Mengganti dependency asli dengan versi palsu yang bisa dikontrol
- [ ] Mempercepat compile
- [ ] Membuat database baru

**4. Integration test berbeda dengan unit test karena...**

- [x] Menguji interaksi antar beberapa modul/komponen
- [ ] Hanya test UI
- [ ] Tidak perlu assertions
- [ ] Lebih cepat dari unit test

**5. CI/CD kepanjangan dari...**

- [ ] Code Integration / Code Deployment
- [x] Continuous Integration / Continuous Deployment
- [ ] Computer Interface / Computer Design
- [ ] Code Installation / Code Development

**6. GitHub Actions adalah...**

- [ ] Database services
- [x] Platform CI/CD terintegrasi dengan GitHub
- [ ] Code editor
- [ ] Package manager

**7. Assertion `expect(result).toBe(5)` akan pass jika...**

- [ ] result adalah string "5"
- [x] result === 5 (strict equality)
- [ ] result == 5 (loose)
- [ ] result tidak null

**8. Test coverage mengukur...**

- [x] Persentase kode yang tercakup oleh test
- [ ] Jumlah test yang gagal
- [ ] Kecepatan test
- [ ] Ukuran file test

**9. `describe` dan `it` pada Jest digunakan untuk...**

- [ ] Membuat loop
- [x] Mengelompokkan dan mendefinisikan test cases
- [ ] Mengimport modul
- [ ] Menjalankan script

**10. Manfaat utama CI/CD pipeline?**

- [ ] Menambah dependency
- [x] Otomatisasi testing dan deployment setiap ada perubahan kode
- [ ] Mengganti bahasa pemrograman
- [ ] Membuat dokumentasi manual
