# 26. Pragmatic Programming

> **Level:** 🟡 Intermediate  
> **Jam:** 8 (4 minggu × 2 sesi)  
> **Prasyarat:** Dasar TypeScript/JavaScript, familiar dengan code editor  
> **Output:** Codebase yang sudah direfactor dengan prinsip-prinsip pragmatic programming

## Tujuan Pembelajaran

Setelah modul ini, kamu bisa:
- Menerapkan prinsip DRY, KISS, YAGNI, Boy Scout Rule, dan SRP dalam kode sehari-hari
- Menulis kode dengan naming convention yang konsisten dan mudah dibaca
- Melakukan refactoring dengan pola-pola yang terbukti (extract function, rename, decompose conditional)
- Memberi dan menerima code review secara konstruktif dengan checklist yang sistematis
- Memperkirakan effort tugas dengan teknik PERT, planning poker, dan reference task
- Menghilangkan magic numbers dan meningkatkan maintainability kode

## AI Prompt Exercises

Sepanjang modul, latihan pake AI:
- "Review this code for DRY violations and suggest improvements"
- "Extract this function into smaller focused functions"
- "Rename these variables following clean code conventions"
- "Generate 5 test cases that would catch bugs in this refactored code"
- "Explain why this code violates the Single Responsibility Principle"
