# Quiz: Pragmatic Programming

<div class="quiz">

**1. Prinsip DRY (Don't Repeat Yourself) berarti:**

- [ ] Tidak boleh menggunakan fungsi yang sama dua kali
- [x] Setiap pengetahuan harus memiliki representasi tunggal yang otoritatif dalam sistem
- [ ] Menghapus semua komentar agar kode lebih bersih
- [ ] Menggandakan kode untuk redundansi

**2. Dalam konteks naming, mana nama variabel yang PALING jelas?**

- [ ] `let x = getData()`
- [x] `let userList = fetchActiveUsers()`
- [ ] `let a = db.query()`
- [ ] `let tmp = calc()`

**3. YAGNI (You Ain't Gonna Need It) memperingatkan kita untuk:**

- [x] Tidak menambahkan fungsionalitas sampai benar-benar diperlukan
- [ ] Selalu menulis kode untuk skenario terburuk
- [ ] Membuat arsitektur generik sejak awal
- [ ] Menolak semua perubahan requirement

**4. Dalam code review, komentar mana yang PALING konstruktif?**

- [ ] "Kode ini jelek, bikin ulang."
- [x] "Fungsi `processData` terlalu panjang. Pertimbangkan dipecah menjadi `validateInput`, `transformData`, dan `saveResult`."
- [ ] "Biarin aja, nanti direfactor."
- [ ] "Ini error, cari sendiri."

**5. Teknik estimasi yang menggunakan perbandingan relatif antar task disebut:**

- [ ] Timeboxing
- [ ] Critical Path Method
- [x] Planning Poker / Story Points
- [ ] Gantt Chart

**6. Refactoring sebaiknya dilakukan ketika:**

- [ ] Aplikasi sudah live dan stabil
- [x] Ada code smell, tes sudah lulus, dan perubahan bisa dilakukan secara inkremental
- [ ] Seminggu sebelum deadline besar
- [ ] Hanya saat ada bug yang dilaporkan user
