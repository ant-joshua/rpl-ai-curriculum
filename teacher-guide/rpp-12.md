# RPP: UI/UX Design

| Info | Detail |
|------|--------|
| Kode | RPL-AI-12 |
| Durasi | 4 pertemuan × 90 menit |
| Level | Menengah |
| Prasyarat | Bootstrap/Tailwind atau CSS dasar |

## Pertemuan 1: Design Thinking (Empathize → Define → Ideate)

### Tujuan
- Memahami design thinking process
- Melakukan empathize (user interview/observation)
- Mendefinisikan masalah user dan ideate solusi

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 10' | **Apersepsi**: "Kenapa ada aplikasi yang enak dipake dan ada yang enggak?" — demo side by side | Tanya jawab | Slide + browser |
| 20' | **Design Thinking overview**: 5 tahap — empathize, define, ideate, prototype, test | Ceramah | Slide diagram |
| 15' | **Empathize**: User persona, pain points, empathy map | Ceramah + contoh | Slide template |
| 15' | **Define & Ideate**: Problem statement, ideation, prioritization | Ceramah + contoh | Slide |
| 25' | **Praktik**: Buat empathy map + user persona untuk aplikasi "todo app / catatan keuangan / jadwal pelajaran" | Hands-on | Figma / kertas |
| 5' | **Refleksi**: 1 insight tentang user yang lo dapet | Q&A | — |

### Bahan Ajar
- [Module 12 - Design Thinking](../12-ui-ux-design/01-design-thinking.md)

## Pertemuan 2: UI Principles & Wireframing

### Tujuan
- Menerapkan color theory, typography, layout grid
- Membuat wireframe low-fi → hi-fi di Figma
- Memahami visual hierarchy

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 5' | **Review design thinking** | Show persona | — |
| 15' | **Color theory**: color wheel, psychology, palette, contrast | Ceramah + demo | Slide + coolors.co |
| 10' | **Typography**: font pairing, hierarchy, readability | Ceramah + demo | Slide + Google Fonts |
| 10' | **Layout grid**: column grid, spacing, alignment, consistency | Ceramah + demo | Slide + Figma |
| 15' | **Wireframing**: low-fi → hi-fi, tools, best practices | Ceramah + demo | Figma |
| 25' | **Praktik**: Buat wireframe 3 screen (Login, Dashboard, Detail) di Figma | Hands-on | Figma |
| 10' | **Peer review**: Kritik wireframe teman — 1 suka + 1 saran | Feedback | Figma |

### Bahan Ajar
- [Module 12 - UI & Wireframe](../12-ui-ux-design/02-ui-wireframe.md)

## Pertemuan 3: Prototyping & Design System

### Tujuan
- Membuat interactive prototype di Figma
- Membangun design system mini (5 komponen reusable)
- Mempersiapkan handoff developer

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 5' | **Review wireframe** | Show hasil | — |
| 15' | **Interactive prototype**: Figma prototyping, connections, transitions | Ceramah + demo | Figma |
| 15' | **Design system**: komponen atom → molekul → organism, variant, auto layout | Ceramah + demo | Figma |
| 10' | **Handoff developer**: inspect, export assets, spesifikasi | Ceramah + demo | Figma dev mode |
| 35' | **Praktik**: Buat design system — button, input, card, navbar, icon — + prototype flow end-to-end | Hands-on | Figma |
| 10' | **Refleksi**: Design system = kenapa penting buat tim? | Diskusi | — |

## Pertemuan 4: Accessibility & Portfolio

### Tujuan
- Menerapkan accessibility (WCAG, ARIA, contrast ratio)
- Menulis semantic HTML yang accessible
- Menyusun portfolio UI/UX

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 5' | **Review prototype** | Show hasil | — |
| 15' | **WCAG principles**: Perceivable, Operable, Understandable, Robust | Ceramah | Slide |
| 15' | **ARIA & semantic HTML**: roles, labels, keyboard nav, contrast checker | Ceramah + demo | Live code + axe DevTools |
| 15' | **Portfolio UI/UX**: case study structure, screenshot, proses | Ceramah + contoh | Behance / Dribbble |
| 30' | **Praktik**: Accessibility audit prototype — cek contrast, label, tab order | Hands-on | Figma + axe |
| 10' | **Showcase**: 3 siswa terbaik presentasi prototype + portfolio | Show & tell | Projector |
| 5' | **Refleksi**: Desain yang bagus = desain yang inklusif | Q&A | — |
