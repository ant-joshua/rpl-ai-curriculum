# RPP Modul 15: Agile & Scrum

**Durasi:** 2 sesi × 90 menit = 180 menit

## Tujuan Pembelajaran

Setelah modul ini, siswa mampu:
- Menjelaskan perbedaan Agile vs Waterfall
- Menjalankan 4 Scrum ceremony (Planning, Daily, Review, Retro)
- Menulis User Story INVEST + Acceptance Criteria
- Menggunakan GitHub Projects sebagai Scrum board

## Tools & Bahan

- GitHub Projects
- Post-it / Miro / Jira (opsional)
- Contoh project web sederhana
- Sprint template

---

## Sesi 1: Agile Mindset + Scrum Framework (90 menit)

| Waktu | Aktivitas | Detail |
|-------|-----------|--------|
| 15 menit | **Teori: Agile vs Waterfall** | Waterfall rigid, Agile iteratif. Manifesto Agile. 3 Scrum roles (PO, SM, Dev). |
| 45 menit | **Coding: Setup GitHub Projects** | Bikin repo, setup Projects board (To Do, In Progress, Done). Bikin Sprint backlog. Role play: simulasi sprint planning. |
| 20 menit | **Latihan: User Story Writing** | Siswa tulis 5 user story untuk project (todo app / e-commerce). Format INVEST + Acceptance Criteria. |
| 10 menit | **Review** | Kenapa Agile cocok untuk software? Bedanya Product Owner vs Scrum Master? |

**Template user story:**

```markdown
**Sebagai** pengguna,
**saya ingin** login dengan Google,
**agar** tidak perlu register manual.

**Acceptance Criteria:**
- [ ] Tombol "Login with Google" tersedia
- [ ] Redirect ke dashboard setelah sukses
- [ ] Email tercatat di database
```

**Checklist siswa:**
- [ ] GitHub Projects board aktif
- [ ] 5 user story INVEST-compliant
- [ ] Sprint backlog dengan story points
- [ ] Role assignment (PO, SM, Dev)

## Sesi 2: Sprint Execution + Retrospective (90 menit)

| Waktu | Aktivitas | Detail |
|-------|-----------|--------|
| 15 menit | **Teori: Sprint Cycle** | Daily Standup (15 menit, 3 questions). Burndown chart. Sprint Review (demo). Retrospective (Start/Stop/Continue). |
| 45 menit | **Coding: Sprint Simulation** | Role play: 2 sprint cycle (15 menit per sprint). Daily standup tiap 5 menit. Review + demo hasil. Retro dengan action items. |
| 20 menit | **Latihan: Sprint Retro + Burndown** | Siswa bikin burndown chart di GitHub. Tulis action items retro. |
| 10 menit | **Review** | Refleksi: apa blocker terbesar? Gimana improve next sprint? |

**Template daily standup:**

```text
1. Apa yang saya kerjakan kemarin?
2. Apa yang saya kerjakan hari ini?
3. Ada blocker?
```

**Template retro:**

```markdown
## Start (mulai lakukan)
- [ ] Code review sebelum merge
## Stop (berhenti lakukan)
- [ ] Daily standup molor
## Continue (lanjutkan)
- [ ] Pair programming
```

**Checklist siswa:**
- [ ] Sprint planning dengan Sprint Goal
- [ ] Daily standup ≤ 15 menit
- [ ] Sprint review dengan demo
- [ ] Retro dengan action items konkret
- [ ] Burndown chart ter-update
