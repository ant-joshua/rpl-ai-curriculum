## Pertemuan 1: Internet, Client-Server & HTTP

### Tujuan
- Memahami konsep client-server dan cara internet bekerja
- Membedakan frontend vs backend
- Memahami struktur URL, HTTP methods, dan status codes

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 10' | **Apersepsi**: Tanya "gimana internet bekerja?" — tulis asumsi siswa di papan | Brainstorming | Whiteboard |
| 20' | **Client-Server**: Analogi restoran, diagram request-response, contoh IG/FB | Ceramah + visual | Slide, diagram |
| 15' | **Frontend vs Backend**: Demo buka IG — inspeksi elemen vs network tab | Demo live | Browser devtools |
| 10' | **HTTP & URL**: Breakdown URL, HTTP methods & status codes | Ceramah | Slide + kartu status code |
| 15' | **Aktivitas kelompok**: Tebak HTTP method & status code dari skenario | Game | Kartu soal |
| 15' | **Coba langsung**: `curl` request ke dummy API, lihat response | Hands-on | Terminal, https://jsonplaceholder.typicode.com |
| 5' | **Refleksi**: 3 hal baru yang dipelajari hari ini | Tanya jawab | — |

## Pertemuan 2: API, Database & Deployment

### Tujuan
- Memahami konsep API sebagai jembatan antar-aplikasi
- Mengetahui peran database dalam aplikasi
- Memahami gambaran umum deployment

### Kegiatan (90 menit)

| Waktu | Aktivitas | Metode | Media |
|-------|-----------|--------|-------|
| 10' | **Review**: Recap client-server, HTTP methods | Q&A cepat | Slide |
| 20' | **API**: Analogi pelayan restoran, demo pake public API (cuaca/IG) | Ceramah + demo | Postman/curl |
| 10' | **Database**: Analogi lemari arsip, SQL vs NoSQL visual | Ceramah | Slide diagram |
| 15' | **Deployment**: Analogi ngekos-in aplikasi, Vercel/Railway | Ceramah | Browser demo |
| 20' | **Membongkar web**: Siswa buka web favorit, identifikasi FE/BE/API/DB | Praktik kelompok | Browser |
| 10' | **Roadmap**: Gambaran besar kursus — modul 00→12, gimana semuanya nyambung | Ceramah | Peta kurikulum |
| 5' | **Refleksi & pretest**: Survey pengetahuan coding | Kuis | Google Form / kertas |
