## 2. Simulasi Uji Praktik (90 menit)

### Soal Praktik

Buat **Aplikasi Manajemen Kontak** dengan fitur:

```markdown
### Fitur Wajib (Nilai 70%)

1. ✅ Registrasi & Login user (JWT)
2. ✅ CRUD kontak (nama, email, no. telepon, alamat)
3. ✅ Validasi input (nama minimal 3 karakter, format email valid)
4. ✅ Search kontak by nama
5. ✅ Responsive design

### Fitur Bonus (Nilai 30%)

6. 🔷 Kategori kontak (teman, keluarga, kerja)
7. 🔷 Upload foto profil kontak
8. 🔷 Export kontak ke CSV
9. 🔷 Pagination
10. 🔷 Dark mode toggle
```

### Stack yang Boleh Digunakan

| Layer | Pilihan |
|-------|---------|
| Frontend | React / Vue / Vanilla JS |
| Backend | Node.js / PHP / Python |
| Database | MySQL / PostgreSQL / SQLite |
| CSS | Tailwind / Bootstrap / Vanilla CSS |

### Rubrik Penilaian

| Kriteria | Bobot | 0 (Kurang) | 1 (Cukup) | 2 (Baik) | 3 (Sangat Baik) |
|----------|-------|------------|------------|----------|------------------|
| Fungsionalitas | 30% | < 3 fitur jalan | 3-4 fitur | 5 fitur | Semua fitur + bonus |
| Kualitas Kode | 25% | Spaghetti, no structure | Ada struktur | Modular, clean | Best practices |
| Database | 15% | 1 tabel, no relation | 2 tabel | Normalized 3NF | + indexing, migration |
| Keamanan | 10% | SQL injection risk | Prepared stmt | + validasi | + rate limit, JWT |
| UI/UX | 10% | Tidak responsive | Responsive | + good UX | + animasi, loading |
| Dokumentasi | 10% | No README | README minimal | README lengkap | + API docs, screenshots |

## 3. Simulasi Wawancara (30 menit)

### 3.1 Sesi Tanya Jawab

Asesor akan mengajukan pertanyaan berikut (atau variannya):

| # | Pertanyaan | Area |
|---|-----------|------|

| 3 | "Bagaimana cara kamu handle error di aplikasi?" | Error handling |
| 4 | "Apa tantangan terbesar saat membuat aplikasi ini?" | Problem solving |
| 5 | "Bagaimana aspek keamanan di aplikasi kamu?" | Security |
| 6 | "Kalau ada bug production, bagaimana proses debugging kamu?" | Debugging |
| 7 | "Apa yang akan kamu tingkatkan jika ada waktu 1 minggu lagi?" | Growth mindset |
| 8 | "Bagaimana cara testing aplikasi kamu?" | Testing |

### 3.2 Contoh Jawaban Baik

**Q:** "Bagaimana cara kamu handle error di aplikasi?"

```
"Di aplikasi saya, error handling dilakukan berlapis:

1. **Client-side:** Validasi form dengan Zod sebelum submit.
   Menampilkan pesan error user-friendly.

2. **Server-side:** Global error middleware yang menangkap semua
   error dan mengembalikan response JSON konsisten.

3. **Logging:** Error dicatat ke file log dengan timestamp,
   severity level, dan stack trace untuk debugging.

4. **User feedback:** Semua error ditampilkan sebagai toast
   notification — bukan alert atau console.log.

Menurut saya, error handling yang baik bukan cuma menangkap
error, tapi juga memberikan pengalaman yang baik ke user saat
terjadi kesalahan."
```

### 3.3 Daftar Periksa Wawancara

Sebelum sesi wawancara, pastikan kamu bisa menjelaskan:

- [ ] Arsitektur aplikasi (frontend-backend-database)
- [ ] Alur autentikasi (register → login → JWT → protected routes)
- [ ] Pola keamanan (prepared statement, validasi, CORS)
- [ ] Struktur database & relasi
- [ ] Alur request dari user sampai response
- [ ] Cara deployment (jika ada)
- [ ] Kelemahan aplikasi & rencana perbaikan
- [ ] Pengalaman belajar & tantangan yang dihadapi

## 4. Final Checklist Kesiapan

Gunakan checklist ini untuk evaluasi akhir:

### Dokumen & Administrasi
- [ ] Pendaftaran sertifikasi sudah dikirim
- [ ] Persyaratan administrasi lengkap (KTP, pas foto, ijazah)
- [ ] Biaya sertifikasi sudah dibayar
- [ ] Jadwal uji sudah dikonfirmasi
- [ ] Lokasi uji diketahui (online/offline)

### Pengetahuan Teori
- [ ] Konsep pemrograman dasar (algoritma, struktur data)
- [ ] Web development (HTML, CSS, JavaScript)
- [ ] Framework (React/Vue) & backend (Node.js/PHP)
- [ ] Database (SQL, normalisasi, indexing)
- [ ] Security (OWASP Top 10, SQL injection, XSS)
- [ ] Version control (Git branching, merge, conflict)
- [ ] Deployment & DevOps dasar

### Kesiapan Praktik
- [ ] Laptop & charger siap
- [ ] Environment development sudah terinstall
- [ ] Template/boilerplate project siap
- [ ] Biasakan diri dengan IDE shortcuts
- [ ] Latihan membuat fitur CRUD dalam 30 menit

### Mental & Fisik
- [ ] Istirahat cukup (minimal 7 jam tidur)
- [ ] Sarapan sebelum ujian
- [ ] Datang 30 menit lebih awal
- [ ] Bawa air minum
- [ ] Percaya diri — kamu sudah siap!

## 5. Action Plan Setelah Sesi Ini

### Jika Lulus Sertifikasi

1. **Perbarui CV & LinkedIn** — Tambahkan sertifikasi
