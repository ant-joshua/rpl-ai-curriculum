# 📍 Sesi 01 — Overview Jalur Sertifikasi RPL

> **Durasi:** 120 menit  
> **Tujuan:** Memahami skema sertifikasi RPL (BNSP/LSP), jenis-jenis sertifikasi, dan memilih jalur yang sesuai dengan kompetensi

---

## 1. Mengapa Sertifikasi Penting?

### 1.1 Manfaat Sertifikasi

| Manfaat | Deskripsi |
|---------|-----------|
| **Pengakuan Kompetensi** | Bukti formal bahwa kamu menguasai standar industri |
| **Nilai Tambah CV** | Pembeda saat melamar kerja dibanding kandidat lain |
| **Standar Industri** | Belajar sesuai standar yang diakui dunia kerja |
| **Kenaikan Karir** | Syarat kenaikan jabatan / grade di banyak perusahaan |
| **Kepercayaan Diri** | Validasi eksternal atas kemampuan yang dimiliki |

### 1.2 Sertifikasi vs Portofolio

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   PORTOFOLIO                    SERTIFIKASI                 │
│   ──────────                    ──────────                  │
│   • Bukti karya nyata           • Bukti kompetensi teruji   │
│   • Menunjukkan skill           • Menunjukkan pengetahuan   │
│   • Subjektif (penilai)         • Objektif (standar baku)   │
│   • Butuh waktu lama            • Butuh persiapan terfokus  │
│   • Wajib untuk developer       • Recommended untuk karir   │
│                                                             │
│   💡 Ideal: PORTOFOLIO + SERTIFIKASI = PACKAGE LENGKAP     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. Jenis Sertifikasi RPL

### 2.1 Skema Sertifikasi BNSP / LSP

Di Indonesia, sertifikasi RPL dikelola oleh **LSP (Lembaga Sertifikasi Profesi)** yang terlisensi **BNSP (Badan Nasional Sertifikasi Profesi)**.

| Skema | Level | Cocok Untuk |
|-------|-------|-------------|
| **Junior Programmer** | KKNI Level II-III | Fresh graduate, siswa SMK |
| **Programmer** | KKNI Level III-IV | 1-3 tahun pengalaman |
| **Senior Programmer** | KKNI Level IV-V | 3+ tahun pengalaman |
| **System Analyst** | KKNI Level IV-V | 5+ tahun pengalaman |
| **Database Administrator** | KKNI Level IV | Spesialis database |
| **Web Developer** | KKNI Level III-IV | Full-stack developer |

### 2.2 Skema Internasional

| Sertifikasi | Badan | Fokus |
|-------------|-------|-------|
| **AWS Certified Developer** | Amazon | Cloud development |
| **Google Associate Cloud Engineer** | Google | Cloud engineering |
| **Microsoft Azure Developer** | Microsoft | Azure development |
| **Oracle Certified Java Programmer** | Oracle | Java programming |
| **CompTIA IT Fundamentals** | CompTIA | IT fundamentals |
| **Python Institute Certifications** | OpenEDG | Python programming |

### 2.3 Skema Vendor-Spesifik

| Sertifikasi | Vendor | Teknologi |
|-------------|--------|-----------|
| Meta Front-End Developer | Meta / Coursera | React, HTML, CSS, JS |
| Meta Back-End Developer | Meta / Coursera | Node.js, Python, Django |

## 3. Sertifikasi BNSP RPL — Detail

### 3.1 Unit Kompetensi (Web Developer)

Sertifikasi Web Developer BNSP terdiri dari beberapa unit kompetensi:

| Kode Unit | Unit Kompetensi | Jenis |
|-----------|-----------------|-------|
| J.620100.002.01 | Menggunakan Struktur Data | Dasar |
| J.620100.003.01 | Merancang Database | Inti |
| J.620100.007.01 | Membangun Aplikasi Web | Inti |
| J.620100.008.01 | Mengintegrasikan Komponen Web | Inti |
| J.620100.009.01 | Mengelola Keamanan Aplikasi Web | Inti |
| J.620100.010.01 | Melakukan Pengujian Aplikasi | Pendukung |
| J.620100.011.01 | Menyusun Dokumentasi | Pendukung |
| J.620100.012.01 | Melakukan Deployment Aplikasi | Pendukung |

### 3.2 Skema Uji Kompetensi

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   PROSES UJI KOMPETENSI BNSP                     │
│                                                  │
│   1. Pendaftaran → verifikasi berkas             │
│   2. Asesmen Mandiri → self-assessment           │
│   3. Uji Teori → pilihan ganda & essay           │
│   4. Uji Praktik → proyek pengembangan aplikasi  │
│   5. Wawancara → klarifikasi hasil praktik       │
│   6. Sertifikasi → penerbitan sertifikat BNSP    │
│                                                  │
│   ⏱️ Total proses: 2-5 hari (tergantung skema)  │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 3.3 Metode Uji

| Metode | Bobot | Durasi |
|--------|-------|--------|
| **Tes Tulis** — Pengetahuan konsep, teori, best practice | 30% | 90 menit |
| **Tes Praktik** — Implementasi aplikasi sesuai studi kasus | 50% | 180 menit |
| **Wawancara** — Klarifikasi & pendalaman kompetensi | 20% | 30 menit |

## 5. Memilih Jalur Sertifikasi

### 5.1 Decision Tree

```
Apakah kamu sudah punya pengalaman coding?
│
├── Ya, kurang dari 1 tahun
│   └── → Junior Programmer (BNSP) / CompTIA IT Fundamentals
│
├── Ya, 1-3 tahun
│   └── → Web Developer (BNSP) / AWS Certified Developer
│
├── Ya, 3+ tahun
│   └── → Senior Programmer (BNSP) / Google Cloud Engineer
│
└── Belum ada pengalaman
    └── → Selesaikan kurikulum RPL dulu, lalu ambil Junior Programmer
```

### 5.2 Rekomendasi Berdasarkan Jalur Belajar

| Jalur RPL | Sertifikasi Rekomendasi |
|-----------|------------------------|
| **Full-Stack Web** | Web Developer BNSP / AWS Developer |
| **Frontend Web** | Meta Front-End Developer |
| **Backend API** | Back-End Developer BNSP / Azure Developer |
| **AI Agent** | Google Cloud Engineer + AI certifications |

## 6. Latihan: Assessment Plan

### Tugas Individu (30 menit)

1. Gunakan **Self-Assessment Matrix** di atas → evaluasi diri kamu
2. Tentukan **3 unit kompetensi** yang paling perlu kamu pelajari
3. Buat **rencana belajar 4 minggu** untuk 3 unit tersebut
4. Diskusikan dengan teman / mentor

### Template Rencana

```markdown
## Rencana Belajar — [Nama]

### Target Sertifikasi: ________________
### Target Waktu: ________________

### Prioritas 1: [Unit Kompetensi]
- Minggu 1: [topik]
- Minggu 2: [topik]
- Target: [hasil yang dicapai]

### Prioritas 2: [Unit Kompetensi]
- ...
```

### Kriteria Sukses

- [ ] Self-assessment terisi lengkap
- [ ] 3 prioritas belajar teridentifikasi
- [ ] Rencana 4 minggu realistic
- [ ] Target sertifikasi jelas
