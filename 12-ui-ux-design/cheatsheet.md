# 🧠 Cheatsheet: UI/UX Design

> Referensi cepet — 1 halaman. Print atau bookmark.

## Topik Utama
- **Design Thinking**: Empathize → Define → Ideate → Prototype → Test
- **Wireframing**: Low-fi (sketsa) → Mid-fi (layout jelas) → Hi-fi (detail + warna)
- **Figma**: Tool utama — bikin wireframe, prototype, design system
- **UI Principles**: Color theory, typography, layout grid, spacing, hierarchy
- **Design System**: Komponen reusable (button, input, card, navbar, icon)
- **Accessibility**: WCAG, ARIA labels, contrast ratio, semantic HTML
- **Portfolio**: Case study UX — problem → process → solution

## Color & Typography Quick Ref

- **Color palette**: Primary (brand), Secondary, Accent, Neutral/Gray, Error/Success/Warning
- **WCAG AA**: Contrast ratio ≥ 4.5:1 untuk text normal, ≥ 3:1 untuk large text
- **Typography**: `rem` for font size (1rem = 16px), line-height 1.5 untuk body
- **Spacing system**: 4px scale — 4, 8, 12, 16, 20, 24, 32, 40, 48, 64

## Tips & TriK
- Mulai dari low-fi wireframe di kertas — cepat, murah, iterasi
- Auto Layout di Figma = flexbox — stretch, wrap, gap
- Design system = komponen + variants (size, color, state)
- Interactive prototype: sambungin frame pake prototype tab → simulate flow
- Accessibility: test pake screen reader (VoiceOver / NVDA)

## Common Mistakes
- ❌ Langsung hi-fi tanpa wireframe — byk revisi di tahap mahal
- ❌ Typography inconsistency — batasi 2-3 font family doang
- ❌ Lupa mobile first → layout broken di HP
- ❌ Contrast ratio terlalu rendah → text ga kebaca
- ❌ Prototype tanpa flow logic → user bingung
