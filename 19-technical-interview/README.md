## Ringkasan: Behavioral Interview — Metode STAR

Wawancara perilaku bobotnya setara dengan coding. Gunakan STAR.

| Huruf | Arti | Contoh |
|-------|------|--------|
| **S**ituation | Konteks | "Dalam proyek kelompok akhir semester..." |
| **T**ask | Tanggung jawab | "Saya bertugas bangun REST API dan integrasi database." |
| **A**ction | Langkah konkret | "Saya desain skema, buat endpoint dengan Express, tulis unit test." |
| **R**esult | Dampak | "API selesai 3 hari lebih cepat. Nilai proyek A." |

**Pertanyaan umum:** "Ceritakan kegagalan terbesar." / "Pernah konflik tim?" / "Kenapa kami harus hire kamu?"  
→ Siapkan 5 jawaban STAR. Rekam pakai HP. Evaluasi.

## Ringkasan: CV & LinkedIn

| Area | Checklist |
|------|-----------|
| **CV** | PDF simpel (ATS-friendly), keyword dari job description, 2-3 proyek dengan link |
| **LinkedIn** | Foto profesional, headline jelas, about section, skills + endorsement |
| **Portfolio site** | Hero, tentang, proyek (card + link), skill, kontak |

**Kesalahan fatal CV:** typo nama perusahaan, email tidak profesional, klaim skill tidak relevan.

## Ringkasan: DSA Coding — 5 Pola Utama

| Pola | Kapan Pakai | Kompleksitas Khas |
|------|------------|-------------------|
| **Sliding Window** | Subarray/substring optimal (maks/min) | O(n) |
| **Two Pointers** | Array terurut, cari pasangan | O(n) |
| **Fast & Slow Pointer** | Linked list cycle, middle node | O(n) |
| **BFS/DFS** | Pencarian jalur, level order, shortest path | O(V+E) |
| **DP** | Overlapping subproblems, min/max/count | O(n) / O(n²) |

**Tips:** NeetCode 150. 20-30 menit/soal. Jika stuck: lihat solusi, pahami, ulangi 2 hari kemudian.

## Ringkasan: System Design — Metode 1-2-3

| Langkah | Durasi | Aktivitas |
|---------|--------|-----------|
| **1. Klarifikasi** | 3-5 menit | Tanya kebutuhan, pengguna, prioritas |
| **2. High-Level** | 5-7 menit | Gambar arsitektur + komponen utama |
| **3. Deep Dive** | 10-15 menit | DB schema, API design, data flow |

**Contoh soal untuk fresh grad:** URL shortener, chat app, parking lot, todo API, rating system.

## Ringkasan: Live Coding Strategy

1. Baca soal 2x → tanya clarifying questions → buat contoh  
2. Sebut brute force + kompleksitas  
3. Brainstorm optimasi → tulis pseudocode  
4. Koding — syntax rapi, variable jelas  
5. Test dengan contoh + edge cases  
6. Sebut kompleksitas akhir + trade-off  

**Paling penting:** *Think out loud sepanjang waktu.*

## Ringkasan: Portofolio Walkthrough

Struktur 5 menit: judul → tech stack → arsitektur → fitur utama → tantangan → pelajaran.

**Checklist final:**
- [ ] README proyek lengkap (stack, fitur, screenshot, cara jalanin)
- [ ] Live demo bisa diakses (Vercel/Netlify/Railway)
- [ ] GitHub rapi (commit message, .gitignore, repositori bersih)

## Ringkasan: Mock Interview, Negosiasi & Follow-Up

| Tahap | Tindakan |
|-------|----------|
| **Mock** | Latihan 30 menit dengan teman. Evaluasi pakai tabel skor. |
| **Negosiasi** | Gaji, training, perangkat — riset dulu, bawa data. |
| **Follow-up** | Thank you email (1 jam) → follow-up (3 hari) → feedback (jika ditolak). |
